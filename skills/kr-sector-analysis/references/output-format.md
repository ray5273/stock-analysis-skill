# Output Format

## Default Quick Brief

Use this shape for `quick brief`:

1. 한줄 결론
2. 섹터 정의
3. 핵심 드라이버 3-5개
4. 핵심 리스크 3-5개
5. 대표 상장사와 포지셔닝
6. 지금 볼 숫자와 이벤트
7. 소스

## Default Full Report

Use this shape for `full report`:

```md
# <섹터명>

기준일: YYYY-MM-DD

## 소개 및 연구 가정
## 연구 범위와 방법론
## 행정/정책 개요
## 시장 풍경
## 성장 드라이버와 제약
## 가치사슬
## 규제 프레임워크
## 기술 변화
## 경쟁 구도
## 대표 상장사/비상장 플레이어 맵
## 시장 기회와 미래 전망
## Sources
```

### Full Report Rules

- Add exact dates to policy events, metrics, and company facts.
- If a subsection has weak data, keep it short and say why.
- Use tables sparingly and only where they improve scanability.
- Keep the `Sources` section dated and link-based.

## Tone

- Be direct and analytical.
- Distinguish verified facts from inference.
- Use `not separately disclosed` when the source set does not support a clean number.
