사용자가 `수주계약건만`, `단일판매ㆍ공급계약만`, `최근 1년 계약공시 리스트`처럼 계약공시만 요청하면 아래 형식을 우선 사용한다.

섹션명과 표 컬럼은 특별한 요청이 없으면 그대로 유지한다.

```md
# <회사명> 수주계약 공시 리스트

## 범위
- 회사명:
- 티커:
```

## 작성 규칙

- 중요한 숫자는 반드시 공시 본문에 연결한다.
- 기준이 다르면 표 안에서 보이게 쓴다.
- 사용자가 `매출 비중`, `구성 비중`, `계약 비중`, `매출 비중과 계약 비중 같이`를 물으면 `## 매출 비중 요약`을 기본으로 채운다.
- `매출 비중 요약`에는 가능하면 아래 항목을 우선 검토한다.
  - `사업부 / 연결 매출 비중`
  - `핵심 세부사업 / 상위 사업부 매출 비중`
  - `주요 고객 / 연결 매출 비중`
  - `특수관계자 / 연결 매출 비중`
  - `대규모기업집단 계열회사 / 연결 매출 비중`
- `수주잔고 및 커버리지` 섹션에서 계약 비중을 같이 다룰 때는 아래 항목을 우선 검토한다.
  - `공식 backlog 숫자`
  - `공식 backlog / 연간 매출`
  - `특정 계약군 수주잔고 / 연결 매출 단순 비중`
  - `최근 공시 계약금액 / 최근매출액대비`
- 계약 관련 비율은 `공식 backlog`, `특정 계약군 수주잔고`, `개별 계약공시`를 구분해서 라벨링한다. 하나의 `계약 비중`으로 뭉뚱그리지 않는다.
- 사용자가 `특수관계자`, `관련당사자`, `내부그룹 매출`, `계열 매출 비중`을 물으면 `재무상태, CAPEX, 특수관계자` 또는 `지역, 고객, 수주 관련` 섹션에 아래 항목을 우선 넣는다.
  - `특수관계자 매출`
  - `대규모기업집단 계열회사 매출`
  - `내부그룹 매출 비중(엄격 기준)`
  - `내부그룹 매출 비중(확장 기준)`
  - `고객집중도` 또는 `10% 이상 외부고객` 문구가 있으면 별도 행으로 유지
- `내부그룹 매출 비중`은 가능하면 `엄격 기준`과 `확장 기준`을 둘 다 보여준다. 둘 중 하나만 가능한 경우에만 단일 값으로 쓴다.
- 관련당사자 비중 계산에는 같은 기간의 연결 `매출액`을 분모로 사용하고, note 번호와 계산식을 `비고`나 `Source Map`에 남긴다.
- 다부문 기업에서는 `사업부문 및 세부 구성` 안에 `사업부 개요`, `사업부 실적 매트릭스`, `세부 구성 및 해석 포인트` 순서를 우선 사용한다.
- 다부문 기업에서는 약어만 단독으로 쓰지 말고 첫 표에서 `HS (Home Appliance Solution)`처럼 풀네임을 함께 쓴다.
- `사업부 실적 매트릭스`의 열 이름에는 `연간`, `누적`, `4Q 환산`처럼 기준을 직접 드러낸다.
- 숫자 비교용 표와 설명용 표를 분리한다. `부문명 + 기간 + 지표`를 한 셀에 섞어 쓰지 않는다.
- 단일 사업부 기업이거나 법인별 운영현황만 공시되는 경우에는 기존 `공시 블록 | 값 또는 핵심 내용` 표를 유지해도 된다.
- 계약공시 전용 모드에서는 `공시행 전체 리스트`와 `최신 유효 계약 상태`를 분리한다.
- `계약 종료시점 분포`와 `누적 계약 커버리지`는 `최신 유효 계약 상태`를 기준으로 만든다.
- `커버리지 요약`은 `누적 계약 커버리지`의 축약판이며, 최소 `2026년까지`, `2027년까지`, `2028년까지`를 보여준다.
- 이 표는 공식 수주잔고가 아니라 `현재 유효 계약의 만기 분포`일 수 있으므로 라벨을 정직하게 쓴다.
- 계약공시 전용 모드의 메인 금액 컬럼은 `억원` 기준으로 쓴다. 필요하면 정확한 `원` 금액은 `비고`나 본문 메모에 따로 남긴다.
- 추정 보완 대신 `not separately disclosed` 또는 `검색 결과 없음`을 쓴다.
- 사용자가 해석을 요구하지 않으면 해석 메모는 최소화한다.

## DART Reference Digest 형식

긴 `사업보고서`, `감사보고서`, 또는 note-heavy 공시를 정밀 추출할 때는 아래 companion 형식을 함께 쓴다.

```md
# <회사명> DART Reference

## Reference Cache Metadata
- 회사명:
- 티커:
- 기준 문서:
- 공시일:
- reference 기준일:
- 최근 확인일:
- 마지막 반영 공시일:
- TOC 기준 섹션 수:
- 완전 파싱:
- 부분 파싱:
- 누락:
- 재검토 필요:

## 문서 커버리지
| 항목 | 값 |
| --- | --- |

## 파싱 커버리지 요약
| 섹션 | 상태 | 본문 길이 | 표 수 | 숫자 블록 수 |
| --- | --- | --- | --- | --- |

## 핵심 섹션별 요약
| 섹션 | 요약 | 비고 |
| --- | --- | --- |

## 미공시 확인 로그
- `not separately disclosed`로 처리한 항목과 확인 근거

## DART Recheck
| 주장 | 상태 | 확인값 또는 판단 | 출처 섹션 | 비고 |
| --- | --- | --- | --- | --- |

## 다음 업데이트 우선 확인 항목
- ...
```

## 커버리지 검증 규칙

- 긴 사업보고서에서는 `## 파싱 커버리지 요약`을 기본으로 남긴다.
- 상태값은 `parsed`, `partial`, `missing`, `needs_review`를 사용한다.
- `missing` 또는 `needs_review` 상태인 섹션은 미공시 확정이 아니라 재확인 대상으로 취급한다.
- `미공시 확인 로그`에는 어떤 섹션이나 주석을 확인한 뒤 `not separately disclosed`로 판단했는지 남긴다.
- `## DART Recheck`는 memo-critical claim이 있는 annual filing에서는 기본 섹션으로 남긴다.
- 상태값은 `confirmed`, `partially_supported`, `contradicted`, `not_separately_disclosed`, `needs_follow_up`를 사용한다.

## 통합 형식 규칙

- `공식 수주잔고`와 `개별 계약공시 누적액`을 같은 개념처럼 쓰지 않는다.
- `공식 수주잔고 대비 공시 포착 범위`는 단지 최근 12개월에 개별 공시로 관찰된 계약금액의 범위를 뜻한다.
- 금액이 `공개 유보` 또는 `미공개`면 건수는 유지하고 금액 합계에서는 제외한다.
