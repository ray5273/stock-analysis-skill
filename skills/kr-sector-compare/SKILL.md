# Korean Sector Compare

Use this skill for side-by-side Korea sector comparison work.

## Quick Start

- Normalize both sectors to the same date and source hierarchy before comparing.
- Compare only a few metrics that actually matter for the two sectors.
- Explain why the sectors deserve different treatment instead of forcing false symmetry.
- If the workspace is writable and the user wants a reusable artifact, write it to `analysis-example/kr-sector/<sector-a> vs <sector-b>.md`.

## Workflow

1. Define the pair.
   Clarify what each sector label means and where the overlap ends.
2. Set the common comparison frame.
   Use the same source window and same comparison date.
3. Compare the structure.
   Cover demand visibility, regulation, capital intensity, listed exposure quality, and key risks.
4. Rank cautiously.
   End with best structural setup, cleanest listed exposure, and best near-term risk-reward when the evidence supports it.

Read [references/workflow.md](references/workflow.md) for comparison rules.
Read [references/output-format.md](references/output-format.md) for the default output shape.

## Operating Rules

- Keep the comparison on a same-date basis.
- Say where metrics are not directly comparable.
- Avoid turning a compare request into two isolated mini-reports.
