# Output Format

## Default Research Brief

Use this shape unless the user asks for something else:

1. Request Summary
   Restate the user's request in one or two lines.
2. Working Title
   Give the downstream report title you expect to use.
3. Deliverable Mode
   Choose one of `quick brief`, `full report`, `comparison`, `audit`, or `update`.
4. Scope Definition
   State geography, time horizon, value-chain boundary, and whether listed-company coverage is required.
5. In Scope
   List the areas the downstream report must cover.
6. Out of Scope
   List adjacent areas that should be excluded unless new evidence forces them back in.
7. Key Research Questions
   Give 4-8 concrete questions.
8. Required Source Ladder
   State the preferred source order.
9. Proposed Outline
   Provide the section skeleton that the next skill should follow.
10. Handoff Notes
   Record important assumptions and unresolved data risks.

## Tone

- Be short and operational.
- Avoid writing the actual report early.
- Prefer crisp boundaries over broad umbrella phrasing.
